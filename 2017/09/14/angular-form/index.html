#Forms
控件(input, select, textarea)是用户输入数据的一种方法。表单用来把相关的控件组织在一起。
表单和控制器提供了验证服务， 以至于用户在表单提交到后台之前的无效输入被提醒，这种方式验证比起单独在serve端验证的方式提供了更好的用户体验，因为用户可以及时得到反馈纠正错误。请记住，虽然客户端验证在良好的用户体验方面扮演了重要的角色，但它是可以被绕过的，因此不能完全被信任，安全的应用任然需要服务端验证。

Using CSS classes

为了按照表单以及控件的样式，ngModule添加了这些css类

  1. ng-valid: the model is valid
  2. ng-invalid: the model is invalid
  3. ng-valid-[key]: for each valid key added by $setValidity
  4. ng-invalid-[key]: for each invalid key added by $setValidity
  5. ng-pristine: the control hasn’t been interacted with yet
  6. ng-dirty: the control has been interacted with
  7. ng-touched: the control has been blurred
  8. ng-untouched: the control hasn’t been blurred
  9. ng-pending: any $asyncValidators are unfulfilled

Custom model update triggers

通常，任何n内容的改变都会触发一个model的更新或者表单的验证，你可以用ngModelOptions指令只绑定你制定的时间列表，来覆盖这个行为。即，ng-model-options = {uopdateOn: ‘blur’},这样的话只有在失去焦点的时候才会去更新model或者验证表单.你可以用空格在事件列表中分割多个事件。例如 ng-model-options = {updateOn: ‘blur mousedown’}
如果你想要保持默认行为，只是想在触发model更新和表单验证时多加一个事件，那就可以将“default”作为一个制定事件。例如：ng-model-options=”{ updateOn: ‘default blur’ }”

Non-immediate (debounced) model updates

你可以使用ng-model-options的指令的关键字‘debounce’去延迟指令的更新和验证. This delay will also apply to parsers, validators and model flags like $dirty or $pristine

例如： ng-model-options=”{ debounce: 500 }” 将会在改变了内容半秒钟之后去触发model的更新验证。
如果使用自定义触发器，k而已为每一个事件，自定义延迟时间，只需要用一个对象在debounce属性里面。这在某些特定情况下.强制立即更新，是很有用的。例如失去焦点事件。
例如： ng-model-options=”{ updateOn: ‘default blur’, debounce: { default: 500, blur: 0 } }”

如果一个属性被添加进一个element中，那么它的child element和控件也会继承它。除非你复写了。

这个例子展示了，如何debouncem模型的改变,model将会在250毫秒后更新。
